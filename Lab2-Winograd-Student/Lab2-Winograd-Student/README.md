# Lab 2: GEMM / Winograd

这份 README 是对 `Lab2-slides.pdf` 的中文翻译和解释，重点帮助理解本实验为什么要从普通矩阵乘法过渡到 Winograd 卷积。

## 1. 为什么需要更快的矩阵乘法

最朴素的矩阵乘法复杂度是 `O(n^3)`，需要 `n^3` 次标量乘法。  
在现代硬件上，乘法通常比加法更贵，延迟更高、能耗也更大。  
所以一个核心思路是：**能不能用更多加法，换更少乘法**。

### 2x2 矩阵乘法示例

普通算法需要 8 次乘法、4 次加法。  
Strassen 算法（1969）把它变成 7 次乘法、18 次加法，这是第一个突破 `O(n^3)` 的经典算法。

## 2. Strassen 算法

Strassen 的做法是把大矩阵分成 4 个子块，然后构造若干中间量 `S1 ... S7`，最后组合出结果矩阵 `C`。

它的关键点有两条：

1. 每一层递归只做 7 次子矩阵乘法，而不是 8 次。
2. 这种“少一次乘法”的收益会随着递归被不断放大。

### 复杂度

- 朴素算法：每层 8 次子矩阵乘法，整体是 `O(n^3)`
- Strassen：每层 7 次子矩阵乘法，整体是 `O(n^(log2 7)) ≈ O(n^2.807)`

所以矩阵越大，节省就越明显。

## 3. 从矩阵乘法到卷积

Strassen 的思想是：**通过代数重构减少乘法次数**。  
这个思想也能用于卷积运算。

对于 CNN 来说，卷积层通常占据绝大多数计算量，尤其是 `3x3` 卷积核。  
Winograd 算法可以在变换域里做卷积，从而显著减少乘法次数。  
例如 `F(2x2, 3x3)` 可以把乘法量最多降低到直接卷积的约 `1/2.25`。

这也是 cuDNN、TVM、NNPACK 等库会使用 Winograd 的原因。

## 4. `im2col`：把卷积变成矩阵乘法

`im2col` 的思想是把卷积窗口展开成矩阵的一行。

以 1D 为例，输入 `[d0, d1, d2, d3]`，卷积核 `[k0, k1, k2]`，stride 为 1。  
滑动窗口会变成一个矩阵，每一行对应一个 receptive field。  
2D 情况下，把每个 `3x3` patch 展平为一行，就能把卷积转换成一次 GEMM。

优点是可以直接利用高度优化的矩阵乘法实现。  
缺点是会复制大量重叠元素，带来额外内存开销。

## 5. Winograd 卷积

Winograd 的基本形式可以写成：

`Y = A^T [ (G g G^T) ⊙ (B^T d B) ] A`

其中：

- `g` 是卷积核
- `d` 是输入 tile
- `G g G^T` 是卷积核变换
- `B^T d B` 是输入变换
- `⊙` 表示逐元素乘法
- `A^T (·) A` 是逆变换，把结果还原到空间域

### 直观理解

1. 先把卷积核变到 Winograd 域。
2. 再把输入 tile 变到同一个域。
3. 中间只做逐元素乘法，这是整个过程中最少乘法的部分。
4. 最后再变换回输出结果。

对 `F(2x2, 3x3)` 来说，输入 tile 通常是 `4x4`，输出是 `2x2`。

## 6. Tiling 策略

真实特征图通常比 `4x4` 大得多，所以要把它切成多个 `4x4` tile。

因为 `3x3` 卷积核对应的输出步长是 2，所以相邻 tile 之间需要重叠 2 行/2 列，才能无缝拼接输出。

每个 tile 都复用同一组变换矩阵 `A`、`B`、`G`。

## 7. 例子说明

slides 最后给了一个 `4x4` 输入 tile 和 `3x3` 卷积核的验证例子。  
先直接做卷积，得到输出：

`[30, 35; 50, 55]`

然后要求你自己计算：

1. `B^T d B`
2. 与 `G g G^T` 做逐元素乘法
3. 再应用 `A^T (·) A`

最后应该得到和直接卷积相同的结果：

`[30, 35; 50, 55]`

## 8. 本实验的核心结论

这组 slides 想表达的主线是：

1. 乘法比加法贵，所以要尽量减少乘法。
2. Strassen 证明了“用更多加法换更少乘法”是可行的。
3. 同样的思想可以迁移到卷积。
4. Winograd 利用变换域和逐元素乘法，能显著加速 `3x3` 卷积。
5. 代价是实现更复杂，并且要处理 tile 切分、重叠和数值误差等问题。
